# runbook-grounding-agent

A disciplined runbook Q&A system that answers only from retrieved markdown runbooks, always returns citations, and abstains when evidence is insufficient.

## Why It Matters

Incident workflows break when assistants invent procedures. This project enforces a safety-first pattern:

- Retrieval first
- Deterministic answer policy second
- Optional model polishing last
- Explicit abstention when evidence is weak

## MVP Features

- 12 synthetic runbooks across common operations topics
- 25 benchmark questions (`data/benchmarks/benchmark_questions.json`)
- Deterministic retrieval modes:
  - `keyword` (default)
  - `embedding` (deterministic pseudo-embedding mode)
- Structured output:
  - `answer`
  - `cited_chunks[]`
  - `confidence`
  - `insufficient_evidence`
  - `safe_next_steps[]`
  - `related_runbooks[]`
- Web UI with question form, citations panel, answer panel, insufficiency warning
- API endpoints and SQLite-backed query history
- Mock mode default (no OpenAI key required)

## Architecture

```text
[Question + Optional Service Filter]
                 |
                 v
         [Chunker / Corpus]
                 |
                 v
      [Deterministic Retrieval]
      - keyword BM25-like
      - optional embedding-like
                 |
                 v
       [Deterministic Answer Policy]
      - cite evidence
      - confidence score
      - abstain if weak evidence
                 |
                 v
    [Optional OpenAI Answer Polisher]
                 |
                 v
      [Persist + API/Web Response]
```

## Local Setup

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
uvicorn app.main:app --reload
```

## Environment Variables

See `.env.example`.

Key settings:

- `DATABASE_URL`
- `PROVIDER_MODE` (`auto`, `mock`, `openai`)
- `OPENAI_API_KEY` (optional)
- `RETRIEVAL_MODE` (`keyword`, `embedding`)
- `MIN_CONFIDENCE`
- `MAX_CHUNKS`

## Mock Mode

Default mode works fully without `OPENAI_API_KEY`.

- Retrieval and abstention remain deterministic
- Final answer is generated by deterministic policy

## Optional OpenAI Mode

OpenAI is backend-only and never exposed to browser code.

```bash
export PROVIDER_MODE=openai
export OPENAI_API_KEY=your_key
export OPENAI_MODEL=gpt-5-mini
```

## Heroku Deployment

Included files:

- `Procfile`
- `runtime.txt`
- `requirements.txt`
- `.env.example`

Deploy quick path:

```bash
heroku create runbook-grounding-agent
heroku config:set PROVIDER_MODE=mock
heroku config:set RETRIEVAL_MODE=keyword
git push heroku main
```
